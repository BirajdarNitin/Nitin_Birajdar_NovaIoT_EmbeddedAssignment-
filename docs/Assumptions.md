# Assumptions

The following assumptions were made during the design process.

## Business Assumptions

* Each container contains temperature-sensitive products.
* Containers have a stable power source.
* Internet connectivity may not always be available.

## Environmental Assumptions

* Operating temperature range is between -10°C and 50°C.
* Target pharmaceutical storage temperature is 2°C to 8°C.
* Humidity should remain between 40% and 60%.

## Deployment Assumptions

* One device will be installed per container.
* Field technicians can physically access the device if required.
* Thousands of devices may eventually be deployed nationwide.

## System Assumptions

* Local processing takes priority over cloud processing.
* Cloud connectivity is optional in the MVP phase.
* Device configuration changes are infrequent.

## User Assumptions

The system will be used by:

* Drivers
* Field technicians
* Operations managers
* Non-technical personnel

Therefore, alerts must be simple and easy to understand.

## Security Assumptions

* Devices will eventually connect to cloud infrastructure.
* Secure communication will be implemented in future versions.
* Each device will have a unique identity.
